## Descripción
Centralizar y versionar el código de todas las entregas del grupo.

## Lenguajes y herramientas utilizadas

### Lenguajes
- Java
- SQL

### Herramientas y tecnologías
- Git (control de versiones)
- JUnit (pruebas unitarias)
- JDBC (persistencia en base de datos)
- ORM (mapeo objeto-relacional)
- Google Sites (memoria digital del proyecto)

## Contenidos del proyecto

El proyecto se estructura en los siguientes bloques:

### Bloque 1: Diseño UML
Realización de diagramas de casos de uso y diagramas de clases para representar el comportamiento y el diseño estático del sistema.

### Bloque 2: Implementación en Java
Implementación del modelo de clases en Java, uso de estructuras dinámicas de datos, aplicación del patrón MVC, pruebas unitarias con JUnit y uso de Git como sistema de control de versiones.

### Bloque 3: Persistencia con JDBC
Aplicación de persistencia en bases de datos mediante JDBC implementando los patrones MVC, DAO y Factory.

### Bloque 4: Persistencia con ORM
Persistencia mediante mapeo ORM aplicando los patrones MVC, DAO y Factory.
